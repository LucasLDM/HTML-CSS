	<< Hospedando sites com GitHub Pages >>

Sim, é possível e é bem útil pra mostrar para amigos, clientes, etc. Para isso, siga os passos:

1) Vá na sua página do github e selecione o repositório de html-css.

2) Clique em Settings e role até achar GitHub Pages ou apenas Pages (com a atualização, fica do lado esquerdo, agora.)

3) Em branch, selecione "main" e "/root". E clique em Change theme.

4) Pode escolher qualquer tema. Recomendo o Minimal Theme.

5) Fazendo isso, vá até a parte de GitHub Pages de novo em Settings e clique no link que estará disponível. Talvez demore um pouco pra abrir.

O seu site já está no ar! 

*OBS: O conteúdo que aparecerá lá é o mesmo que está no arquivo README.md.

Caso queira acessar um arquivo, vá na URL e depois link "https://lucasldm.github.io/html-css/", insira o caminho até ele. 

No caso foi "https://lucasldm.github.io/html-css/exercicios/desafios/desafio%20010-sideB/android2.html".

----------------------------------------------

Outra dica: Ao invés de acessar sempre pelas URLs, vá no GitHub Desktop e abra o a pasta html-css com o VS Code. Em README.md, você pode alterar o que vai aparecer no site. No caso criei um link chamado "Acessar Site Android" que leva direto pra este site. É mais fácil pra você e também pro cliente, empregador, etc.

*OBS: É normal que demore para aparecer as informações na sua página, mas vão aparecer. E no VS Code, no README.md não tem o autocomplete.

----------------------------------------------

Mais uma dica: Para deixar as coisas mais organizadas, faça o seguinte:

1) Vá no GitHub Desktop e crie um novo repositório público e o caminho dele deve ser na pasta principal, que no caso é esta: "C:\Users\fabia\OneDrive\Documentos\Estudos\Tecnologia\(HTML e CSS) Visual Studio Code\Estudos".

2) Feito isso, pegue as pastas e arquivos importantes do projeto android que está na pasta desafios e em desafio010-sideB copiando ou recortando e inserindo na pasta "projeto-android" no caso.

3) O GitHub Desktop vai reconhecer as mudanças e agora é só dar um Push Origin.

4) No site, no seu perfil, já vai estar lá o projeto. Procure por Pages e agora, selecione as mesmas opções que antes, "main" e "/root" e salve. Espere um tempo e recarregue a página. O novo link que vai direto para o projeto já vai estar disponível.

Agora você pode mandar pra quem quiser!! Inclusive pode acessar pelo celular!!!

Você também pode adicionar um QR code, usando um site que gera isso e colando o link.

----------------------------------------------

A única limitação do GitHub pages é que só aceita linguagem HTML, CSS e Javascript. Não aceita banco de dados, python, node.js, nada além dos 3 pilares.
